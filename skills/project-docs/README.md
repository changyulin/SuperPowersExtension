# Project Docs

维护项目文档的规范。

## 当前内容

- `skills/project-docs/SKILL.md`：项目文档维护规范，覆盖 README、项目总览、架构、业务规则、编码规范、ADR、模块说明和 how-to。

## 使用方式

- 新增或更新文档前，先查看 `skills/project-docs/SKILL.md`。
- 优先复用现有文档，再补充新文档。
- 只写会稳定影响后续开发、验收和协作的内容。

## 建议结构

- `README.md`
- `docs/project-docs/project-overview.md`
- `docs/project-docs/architecture.md`
- `docs/project-docs/business-rules.md`
- `docs/project-docs/coding-guidelines.md`
- `docs/project-docs/adr/`
- `docs/project-docs/modules/<module>/README.md`
- `docs/project-docs/how-to/`
- `docs/project-docs/assets/`

## 维护原则

- 一页能讲清楚的，不写成长文。
- 文档按主题和模块组织，不按时间线堆叠。
- 文档和实际内容不一致时，以最新验收结果为准。
